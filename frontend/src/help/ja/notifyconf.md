# 通知設定

障害発生時や定期レポートの配信方法、音・コマンドなどのアクションを設定する画面。

## パラメータの説明

* **プロバイダー**
  メール送信サービス。smtp（直接送信）/ Google / Microsoft（OAuth2認証を利用した送信）から選択する。
* **クライアントID**
  OAuth2認証（Google / Microsoft）時に使用するクライアントID。
* **クライアントシークレット**
  OAuth2認証（Google / Microsoft）時に使用するクライアントシークレット。
* **テナント名**
  Microsoft OAuth2認証時に使用するAzureのテナントID（またはテナント名）。
* **メールサーバー**
  送信に使用するSMTPサーバーのアドレスとポート（例: `smtp.example.com:587`）。
* **サーバー証明書をチェックしない**
  オレオレ証明書など、SSL/TLS証明書の検証エラーを無視する場合に有効にする。
* **ユーザー**
  SMTP認証時のユーザーID（またはOAuth2送信元のメールアドレス）。
* **パスワード**
  SMTP認証時のパスワード。
* **件名**
  通知メールの件名プレフィックス。
* **送信元**
  送信元（From）のメールアドレス。
* **宛先**
  通知メールの宛先（To）アドレス。カンマ区切りで複数指定が可能。
* **通知レベル**
  メール通知のトリガーとなる障害の最小重要度レベル（重度、軽度、注意など）。
* **通知間隔**
  障害判定チェックおよびメール通知を行う処理周期（分）。
* **定期レポート**
  毎日定刻にシステム稼働状況の日報（定期レポート）メールを送信する機能の有効化。
* **AI要約**
  定期レポートにLLM（AI）によるシステムログの要約解説を添付する機能の有効化。
* **復帰通知**
  障害から復帰（正常化）した際にもメールで通知する機能の有効化。
* **通知Webhook**
  障害発生時に通知メッセージをHTTP POSTで送信する外部Webhook URL。
* **レポートWebhook**
  定期レポートのデータをHTTP POSTで送信する外部Webhook URL。
* **コマンド実行**
  マップのステータス変化時に実行するOSコマンド。コマンド引数内で変数 `$level`（0:重度, 1:軽度, 2:注意, 3:正常, -1:不明）が使用可能。
* **重度障害時に再生する音**
  マップ状態が「重度障害」になった際にブラウザ上で再生するアラーム音声ファイル。
* **軽度障害時に再生する音**
  マップ状態が「軽度障害」になった際にブラウザ上で再生するアラーム音声ファイル。

## ボタンの説明

* **[テスト送信]**（またはテスト） ： 現在設定されているメール設定でテスト用メールを即座に送信するボタン。
* **[Webhookテスト]** ： 設定されている通知Webhook URLに対してテストデータを送信するボタン。
